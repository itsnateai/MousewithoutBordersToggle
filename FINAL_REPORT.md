## Project Summary

MWBToggle is a lightweight AutoHotkey v2 tray utility that toggles Mouse Without Borders (PowerToys) clipboard and file sharing on/off via hotkey or tray icon click. It solves the problem of quickly disabling clipboard sharing for privacy (passwords, sensitive data) without opening the full PowerToys settings UI. The tray icon reflects current state (green = ON, red = OFF), and an on-screen display appears near the cursor on whichever monitor is active.

## Key Features

- **One-key toggle** — Ctrl+Alt+C hotkey (configurable) toggles ShareClipboard + TransferFile simultaneously
- **Tray icon state** — green ON / red OFF icon with periodic sync to catch external changes
- **Multi-monitor OSD** — ToolTip-based notification appears on the active monitor (not just primary)
- **Pause Sharing submenu** — temporarily disable for 5, 15, or 30 minutes with auto-resume
- **Run at Startup** — tray menu toggle that creates/removes a Windows Startup shortcut
- **About dialog** — version, hotkey, and project URL
- **Toggle confirmation** — optional prompt before each toggle (config-driven)
- **Sound feedback** — optional beep on toggle (high tone for ON, low tone for OFF)
- **INI config support** — hotkey, confirmation, and sound feedback configurable via MWBToggle.ini
- **Robust file writes** — retry loop (3 attempts, 200ms), backup before write, JSON validation before commit
- **MWB process detection** — warns if PowerToys Mouse Without Borders is not running

## Lessons Learned

### MWB Settings File Write Pattern
Mouse Without Borders briefly locks its `settings.json` when it processes changes. Robust AHK scripts that write to MWB's settings must use a retry loop (3 attempts, 200ms delay), a pre-write backup, and JSON structure validation before committing the write. Without these, writes during MWB's lock window silently fail or corrupt the file.

### ToolTip vs TrayTip for Notifications
`TrayTip` always fires on the primary monitor as a Windows toast notification. For multi-monitor setups, `ToolTip` at cursor position is far better UX — the notification appears on whichever screen the user is actively using, and disappears cleanly with `SetTimer(() => ToolTip(), -3000)`. No Windows Notification Center pollution.

### Icon Naming Trap
Hardcoding icon filenames (e.g., `off.ico`) makes it impossible to rename icon files without also editing the script. Use descriptive global variables (`g_icoOn`, `g_icoOff`) initialized at file scope, pointing to the actual filenames. This also makes the icon fallback chain (disk → compiled resource → system) easier to reason about.

### INI Config: Minimal Read Pattern
For small utility scripts, a simple "check if file exists, read each key, skip empty values" pattern beats try/catch boilerplate. MWBToggle's LoadConfig reads 3 keys with empty-string guards and keeps the 30-line total. Only scale up to full IniRead+SaveConfig when settings need to be written back by the app.

### AHK Periodic Sync Timer
When state can change externally (MWB reads/writes settings.json), a low-frequency polling timer (`SetTimer(SyncTray, 5000)`) keeps the tray icon accurate without any event hooks. 5 seconds is imperceptible lag for a toggle state indicator.
